**Ordered selections with repetition**

**Unordered selections with repetition**

I prefer to use equation to understand this case: if we need to put $n$ things in $r$ boxes, then we get a equation:

$$x_1 + \cdots + x_r = n$$

where all $x_i$ are nonnegative integers, so the number of solution is what we want. To solve this problem, we turn the equation into a equation of positive integer variables:

$$y_1 + \cdots +y_r = n + r$$

and then using the idea of **stars and bars**, there are $n+r-1$ positions for us to put bars in, and in total we should put $r-1$ bars to get $r$ parts, so the number of solution is $\binom{n+r-1}{r-1}$.

**pigeonhole principle**

### 31 Degree

**degree**
![alt text](../img/MAT1830/week11/week11-1.png)

> **Handshaking lemma**: # degree = 2 * # edges

Hence the sum of degree is even.

The seven bridges

The key is excepting the first and last vertices, all other vertices must have even degree. i.e. there can be at most 2 vertices of odd degree.

**Euler trail**: Contains every edge exactly once. So a graph with more than two vertices of odd degree cannot contain a Euler trail. And if a graph has odd vertex then it cannot contain closed Euler trail. But *conversely* for a graph of all even vertices, there may not exist a Euler trail. But if we assume the graph is **connected**, the answer is yes.

**bridge**

### 32 Trees

**tree**: connected and has no subgraph as a cycle.

One of the most important property of a tree is that if delete an edge then is not connected, otherwise the tree get a loop, which leads to a contradiciton.

> lemma. # edges = # vertices - 1

**spanning tree**
![alt text](../img/MAT1830/week11/week11-3.png)

So any connected graph has at least one spanning tree. (induct the number of edges.)

> Application. Given every edge a weight and find its spanning tree that has the smallest sum of weight.

**greedy algorithm** (对于这个应用，贪心算法可以找到全局最优解。类似的还有十进制数到二进制数的转换。)

### 33 Trees, queues and stacks

**Queues**: Things go in one end and out the other (“first in first out”). The **head**(leftmost one) is the most important one.

**Breadth first ordering**
![alt text](../img/MAT1830/week11/week11-4.png)

For any connected graph, this ordering algorithm also give a spanning tree. Spanning tree generated in this way can find shortest path for every 2 vertices.

**stack**: insert and remove vertices from the same end of S, called the top of the stack, which corresponding to the **depth first algorithm**. (这个算法有一点一笔画的感觉，画不了了再退回找新路; 这种算法找到的 spanning tree, 两个点之间的距离通常就很长)
