### identifiers and primitive carriers

$\mathsf{Id}$ is a countable infinite set of *indentifiers*. $\mathsf{EnumAtom}$ is a set of *enumeration atoms*. 

>what are these 2 concepts?

We use the **primitive carriers**:

$$\mathbb{B}, \mathbb{Z}, \mathbb{R}$$

To index, we use $\mathsf{IdxAtom}$ to denote the set of all admissible index values, where

$$\mathsf{IdxAtom} = \mathbb{Z} + \mathsf{EnumAtom}$$

i.e. it consists of all intergers and all enumeration atoms.

> 所以这段b话告诉我们，所有index的取值范围。

### Finite Ordered Domain

A **finite ordered domain** is a pair:

$$D = (|D|, \prec_D)$$

where $|D|$ is a finite subset of $\mathsf{IdxAtom}$ and $\prec_D$ is a total order relation. $car(D) := |D|$, $enum(D):= \langle d_1,d_2,\cdots,d_n\rangle$ is the sequence listing elements of $|D|$ in the given order.

Let $\mathsf{Dom}$ be the set of all such kind of finite ordered domain.

>这不就是一个有序集吗，然后把他作为指标集

### Values

The **semantic domain of value** is defined by the *least* solution of the equation:

$$\mathsf{Val} \cong \mathbb{B} + \mathbb{Z} + \mathbb{R} + \mathsf{EnumAtom} + \mathsf{Dom} + \mathsf{Arr}$$

where $\mathsf{Arr} := \{(D,f) | D\in \mathsf{Dom}, f:car(D) \to \mathsf{Val}\}$.

The `+` notation here means disjoint union.

> What is the meaning of **least**? For example, $\mathbb{Z}$ and $\mathbb{Z}$ are both a solution of this equation, and which one is smaller?

**Remark.** This semantic domain has less restriction than the *first surface grammar*. **Static typing** can check the basic ill-formed combinations. This is usual in denotational semantics: the semantic universe is larger than the well-typed syntax.

### Environments

A **dynamic environment** is a finite partial map:

$$\rho \in \mathsf{Env}$$

where $\mathsf{Env}$ is the set of all **finite partial map** from $\mathsf{Id}$ to $\mathsf{Val}$.

### Errors and outcomes

There are some different kinds of **constructors**: $\mathsf{ok}(v), \mathsf{fail}, \mathsf{err}(\epsilon)$, presenting normal evaluation, failure of a ground constraint, and undefinedness or dynamic error, respectively.

### Auxiliary operations

Give some examples of **partial meta-level operations**:

- $\delta_{op}:\mathsf{Val}^n \rightharpoonup \mathsf{Val}$, for each **primitive operator** $op$.
  
for example, take $op = +$, then $+(2,5.5) = 7.5$. But to do this, we need this addition symbol represent `float addition`, so we need to resolve and **coercion** to get the same type of objects for the input of the operator.

<!-- A formal core grammar -->

### Types

$$\iota ::= \textbf{int} | \alpha$$
$$\tau ::= \textbf{bool}|\textbf{int}|\textbf{float}|\alpha|\textbf{set\ of}\ \ \iota|\textbf{array}[\iota]\ \textbf{of\ } \tau$$

where $\alpha$ ranges over enumeration type names. Noticing that we allow **recursive definition**. $\iota$ and $\tau$ sound like labeled sets, where we can use elements in the sets to define other elements in the set, i.e. recursive.

### Semantic Signatures

**Principal denotation functions**: for example,

$$\llbracket e \rrbracket^E : Env \to Eval(Val)$$

where $e$ is an expression, $\llbracket \rrbracket$ means finding the semantic of $e$, and $E$ is a symbol of expression.

For example, if we take $e=x+2$, then we need an environment to get the value of $x$: if the environment maps $x \mapsto 5$, then we will get `ok(7)` through the principal denotation function; if the environment doesn't give any value to $x$, then we will get $\mathsf{Err}$.

### Recommended next refinements

1. typing judgments $\Gamma \vdash e : \tau$

Typing judgments is for static checking, which means under environment $\Gamma$, expression $e$ has type $\tau$. This sentence is called **typing judgment**.

<!-- First we write typing judgment for literals -->

$$\Gamma \vdash n: \mathbf{int}$$
$$\Gamma \vdash b: \mathbf{bool}$$
$$\Gamma \vdash f: \mathbf{float}$$
$$\Gamma \vdash e: \alpha$$

2. well-formedness of declarations and models

Starting from an example declaration: `int x:= e;` To show the declaration `d` is **well-formed**, we need to check that 

- $e$ is of type int because we define $x$ is of type $\mathbf{int}$.
- $x$ is not in environment $\Gamma$, because we do not allow repeated definition.
- all variables in expression $e$ must be in the environment $\Gamma$.

So here is the definition of **well-formedness of declarations**:

$$
\frac{\Gamma \vdash e : \tau \quad x \notin \text{dom}(\Gamma)}{\Gamma \vdash (\tau: x = e) \Rightarrow \Gamma \cup \{x : \tau\}}
$$

Since a model consists of declarations and constraints, so we must define the **well-formedness of constraints**.

$$
\frac{\Gamma \vdash e: \mathbf{bool}}{\Gamma \vdash (\text{constraint}\ e) \Rightarrow \Gamma}
$$

Now we can define the **well-formedness of models** recursively:

Starting from the empty model:

$$
\frac{}{\Gamma \vdash [ ] \Rightarrow \Gamma}
$$

And then do the inductive steps:

$$
\frac{\Gamma \vdash i \Rightarrow \Gamma' \quad \Gamma' \vdash [i_1, \dots, i_n] \Rightarrow \Gamma''}{\Gamma \vdash [i, i_1, \dots, i_n] \Rightarrow \Gamma''}
$$

3. a desugaring map from a larger MiniZinc surface fragment into this core.

The desugaring map $\mathcal{D}$ translates surface syntax to core syntax. For basic expressions, it is defined recursively:

$$
\begin{align*}
\mathcal{D}(c) &= c \quad (\text{for constant } c) \\
\mathcal{D}(x) &= x \quad (\text{for identifier } x) \\
\mathcal{D}(e_1 \text{ op } e_2) &= \mathcal{D}(e_1) \text{ op } \mathcal{D}(e_2)
\end{align*}
$$

4. exact rules for coercions and overloaded primitives

For example, the coercion from $\mathbf{int}$ to $\mathbf{float}$:

- if $\Gamma \vdash e_1 : \mathbf{int}$ and $\Gamma \vdash e_1 : \mathbf{float}$, then to add $e_1$ and $e_2$, we need to coerce $e_1$ to $\mathbf{float}$ first, and then overload the addition operator as `float addition`.

$$
% int add int
\frac{\Gamma \vdash e_1 : \mathbf{int} \quad \Gamma \vdash e_2 : \mathbf{int}}{\mathcal{D}(e_1 + e_2) = \mathcal{D}(e_1) +_{\mathbf{i}} \mathcal{D}(e_2)}
$$
$$
% float add float
\frac{\Gamma \vdash e_1 : \mathbf{float} \quad \Gamma \vdash e_2 : \mathbf{float}}{\mathcal{D}(e_1 + e_2) = \mathcal{D}(e_1) +_{\mathbf{f}} \mathcal{D}(e_2)}
$$
$$
% float add int
\frac{\Gamma \vdash e_1 : \mathbf{float} \quad \Gamma \vdash e_2 : \mathbf{int}}{\mathcal{D}(e_1 + e_2) = \mathcal{D}(e_1) +_{\mathbf{f}} \text{int2float}(\mathcal{D}(e_2))}
$$

5. a precise treatment of local constraints and assertions.

To get **local constraint**, we usually use `let`. For example:
```
let {
    int : x = 5;  
    constraint x > 0;
} in x * x
```

where `int : x = 5;` is `i_1`, `constraint x > 0;` is `i_2`, and `x * x` is `e`. Hence this `let` expression equals to 25.

So by abstracting, we can get this rule:

$$
\frac{\Gamma \vdash [i_1, \dots, i_k] \Rightarrow \Gamma' \quad \Gamma' \vdash e : \tau}{\Gamma \vdash (\mathbf{let} \ \{i_1, \dots, i_k\} \mathbf{\ in\ } e) : \tau}
$$

Now move forward to the **assertions**:

Let's focus on the grammar of `assert()` first. It usually looks like this: `assert(c, m, e)`, where `c` is a boolean value, `m` is a string (error code) and `e` has type $\tau$. 

$$
\frac{\Gamma \vdash c : \mathbf{bool} \quad \Gamma \vdash m : \mathbf{string} \quad \Gamma \vdash e : \tau}{\Gamma \vdash \text{assert}(c, m, e) : \tau}
$$

I think first we should distinguish the variables by **parameters** and **decision variables**. And then we should define a **valuation** to assign every decision variable in the model a value. Next, we should define the **set of all solutions**, which consist of all valuations that satisfying the constraints. For optimisation problems, we should find the minimum solution when the valuation ranges over the solution set.

**type instantiation:**

$$\tau ::= \text{inst} \ \sigma$$
$$\text{inst} ::= \mathbf{par} | \mathbf{var}$$
$$\sigma ::= \mathbf{int}|\mathbf{bool}|\mathbf{float}$$

where $\mathbf{par}$ means parameters and $\mathbf{var}$ means decision variables.

**Valuation:**

For a given model $M$, define the set of decision variables: $V := \{x| \Gamma \vdash x: \mathbf{var} \ \sigma\}$. A valuation is defined as:

$$\nu : V \rightarrow \text{Val}$$

**Solution Set:**

$$
\mathcal{S} = \{ \nu \mid \forall (\mathbf{constraint} \ e) \in M, \text{Eval}(e, \nu \cup \rho_{par}) = \text{true} \}
$$

**Optimisation Function:**
$$
f(\nu) := \text{Eval}(e_{obj}, \nu \cup \rho_{par})
$$

$$
\text{Objective} := \min \{ f(\nu) \mid \nu \in \mathcal{S} \}
$$

在 MiniZinc 核心语义中，需要重载的内置操作符主要分为以下几类：

1.  **算术运算符**：`+`, `-`, `*`, `/`, `div`, `mod`（需区分 `int` 和 `float` 的具体实现）。
2.  **比较运算符**：`=`, `!=`, `<`, `>`, `<=`, `>=`（需支持 `int`, `float`, `bool` 及 `set` 等不同类型的比较）。
3.  **逻辑与聚合函数**：`sum`, `product`, `min`, `max`, `forall`, `exists`（这些函数会根据作用的数组元素类型（如 `int` 或 `float`）解析为不同的原语）。
4.  **集合运算符**：`in`, `subset`, `union`, `inter`, `diff` 等（基于元素类型的不同进行重载）。

简而言之，任何可以同时作用于不同基本类型（如整数和浮点数）或不同实例类型（如 `par` 和 `var`）的操作符都需要重载解析。

#### Arithmetic Operators

First we focus on **Arithmetic Operators**.

For example, the surface syntax `+`. In the core syntax, we have `add_int` and `add_float`.

$$
% int add int
\frac{\Gamma \vdash e_1 : \mathsf{int} \quad \Gamma \vdash e_2 : \mathsf{int}}{\mathcal{D}(e_1 + e_2) = \mathcal{D}(e_1) +_{\mathsf{i}} \mathcal{D}(e_2)}
$$
$$
% float add float
\frac{\Gamma \vdash e_1 : \mathsf{float} \quad \Gamma \vdash e_2 : \mathsf{float}}{\mathcal{D}(e_1 + e_2) = \mathcal{D}(e_1) +_{\mathsf{f}} \mathcal{D}(e_2)}
$$
$$
% float add int
\frac{\Gamma \vdash e_1 : \mathsf{float} \quad \Gamma \vdash e_2 : \mathsf{int}}{\mathcal{D}(e_1 + e_2) = \mathcal{D}(e_1) +_{\mathsf{f}} \text{int2float}(\mathcal{D}(e_2))}
$$

Substrction:
$$
\frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(e_1 - e_2) = \mathcal{D}(e_1) -_i \mathcal{D}(e_2)}
$$

$$
\frac{\Gamma \vdash e_1 : \text{float} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(e_1 - e_2) = \mathcal{D}(e_1) -_f \mathcal{D}(e_2)}
$$

$$
\frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(e_1 - e_2) = \text{int2float}(\mathcal{D}(e_1)) -_f \mathcal{D}(e_2)}
$$

unary negation:

$$
\frac{\Gamma \vdash e : \text{int}}{\mathcal{D}(-e) = \text{neg}_i(\mathcal{D}(e))} \quad
\frac{\Gamma \vdash e : \text{float}}{\mathcal{D}(-e) = \text{neg}_f(\mathcal{D}(e))}
$$

Production:

$$
\frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(e_1 * e_2) = \mathcal{D}(e_1) *_i \mathcal{D}(e_2)}
$$

$$
\frac{\Gamma \vdash e_1 : \text{float} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(e_1 * e_2) = \mathcal{D}(e_1) *_f \mathcal{D}(e_2)}
$$

$$
\frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(e_1 * e_2) = \text{int2float}(\mathcal{D}(e_1)) *_f \mathcal{D}(e_2)}
$$

$$
\frac{\Gamma \vdash e_1 : \text{float} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(e_1 * e_2) = \mathcal{D}(e_1) *_f \text{int2float}(\mathcal{D}(e_2))}
$$

Float division:

$$ \frac{\Gamma \vdash e_1 : \text{float} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(e_1 / e_2) = \mathcal{D}(e_1) /_f \mathcal{D}(e_2)} $$

$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(e_1 / e_2) = \text{int2float}(\mathcal{D}(e_1)) /_f \text{int2float}(\mathcal{D}(e_2))} $$

$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(e_1 / e_2) = \text{int2float}(\mathcal{D}(e_1)) /_f \mathcal{D}(e_2)} $$

$$ \frac{\Gamma \vdash e_1 : \text{float} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(e_1 / e_2) = \mathcal{D}(e_1) /_f \text{int2float}(\mathcal{D}(e_2))} $$

Int division:
$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(e_1 \text{ div } e_2) = \mathcal{D}(e_1) \text{ div}_i \mathcal{D}(e_2)} $$

Mod division:
$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(e_1 \text{ mod } e_2) = \mathcal{D}(e_1) \text{ mod}_i \mathcal{D}(e_2)} $$

$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(e_1 = e_2) = \text{int2float}(\mathcal{D}(e_1)) =_f \mathcal{D}(e_2)} $$

#### Logic Operator and Aggregation Functions

`and`
$$\frac{\Gamma \vdash e_1 : \text{bool} \quad \Gamma \vdash e_2 : \text{bool}}{\mathcal{D}(e_1 \land e_2) = \mathcal{D}(e_1) \land_b \mathcal{D}(e_2)} $$

`or`
$$ \frac{\Gamma \vdash e_1 : \text{bool} \quad \Gamma \vdash e_2 : \text{bool}}{\mathcal{D}(e_1 \lor e_2) = \mathcal{D}(e_1) \lor_b \mathcal{D}(e_2)} $$

`not`
$$ \frac{\Gamma \vdash e : \text{bool}}{\mathcal{D}(\neg e) = \neg_b \mathcal{D}(e)} $$

`forall`
$$ \frac{\Gamma \vdash e : \text{array of bool}}{\mathcal{D}(\text{forall}(e)) = \text{forall}_b(\mathcal{D}(e))} $$

`exist`
$$ \frac{\Gamma \vdash e : \text{array of bool}}{\mathcal{D}(\text{exists}(e)) = \text{exists}_b(\mathcal{D}(e))} $$

`int_array_sum`
$$ \frac{\Gamma \vdash e : \text{array of int}}{\mathcal{D}(\text{sum}(e)) = \text{sum}_i(\mathcal{D}(e))} $$

`float_array_sum`
$$ \frac{\Gamma \vdash e : \text{array of float}}{\mathcal{D}(\text{sum}(e)) = \text{sum}_f(\mathcal{D}(e))} $$

mixed
$$ \frac{\Gamma \vdash e : \text{array of int}}{\mathcal{D}(\text{float\_sum}(e)) = \text{sum}_f(\text{map(int2float, } \mathcal{D}(e)))} $$

`int_product`
$$ \frac{\Gamma \vdash e : \text{array of int}}{\mathcal{D}(\text{product}(e)) = \text{product}_i(\mathcal{D}(e))} $$

`float_product`
$$ \frac{\Gamma \vdash e : \text{array of float}}{\mathcal{D}(\text{product}(e)) = \text{product}_f(\mathcal{D}(e))} $$

`min`
$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(\text{min}(e_1, e_2)) = \text{min}_i(\mathcal{D}(e_1), \mathcal{D}(e_2))} $$
$$ \frac{\Gamma \vdash e_1 : \text{float} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(\text{min}(e_1, e_2)) = \text{min}_f(\mathcal{D}(e_1), \mathcal{D}(e_2))} $$

$$ \frac{\Gamma \vdash e : \text{array of int}}{\mathcal{D}(\text{min}(e)) = \text{min}_i^{\text{arr}}(\mathcal{D}(e))} $$
$$ \frac{\Gamma \vdash e : \text{array of float}}{\mathcal{D}(\text{min}(e)) = \text{min}_f^{\text{arr}}(\mathcal{D}(e))} $$

`max`
$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{int}}{\mathcal{D}(\text{max}(e_1, e_2)) = \text{max}_i(\mathcal{D}(e_1), \mathcal{D}(e_2))} $$
$$ \frac{\Gamma \vdash e_1 : \text{float} \quad \Gamma \vdash e_2 : \text{float}}{\mathcal{D}(\text{max}(e_1, e_2)) = \text{max}_f(\mathcal{D}(e_1), \mathcal{D}(e_2))} $$

$$ \frac{\Gamma \vdash e : \text{array of int}}{\mathcal{D}(\text{max}(e)) = \text{max}_i^{\text{arr}}(\mathcal{D}(e))} $$
$$ \frac{\Gamma \vdash e : \text{array of float}}{\mathcal{D}(\text{max}(e)) = \text{max}_f^{\text{arr}}(\mathcal{D}(e))} $$

#### Set Operators

`\in`

`int`
$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{set of int}}{\mathcal{D}(e_1 \in e_2) = \mathcal{D}(e_1) \in_i \mathcal{D}(e_2)} $$

`float`
$$ \frac{\Gamma \vdash e_1 : \text{float} \quad \Gamma \vdash e_2 : \text{set of float}}{\mathcal{D}(e_1 \in e_2) = \mathcal{D}(e_1) \in_f \mathcal{D}(e_2)} $$

mixed
$$ \frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{set of float}}{\mathcal{D}(e_1 \in e_2) = \text{int2float}(\mathcal{D}(e_1)) \in_f \mathcal{D}(e_2)} $$

`\subset`

`int_subset`
$$ \frac{\Gamma \vdash e_1 : \text{set of int} \quad \Gamma \vdash e_2 : \text{set of int}}{\mathcal{D}(e_1 \subseteq e_2) = \mathcal{D}(e_1) \subseteq_i \mathcal{D}(e_2)} $$

`float_subset`
$$ \frac{\Gamma \vdash e_1 : \text{set of int} \quad \Gamma \vdash e_2 : \text{set of float}}{\mathcal{D}(e_1 \subseteq e_2) = \text{set\_i2f}(\mathcal{D}(e_1)) \subseteq_f \mathcal{D}(e_2)} $$

union

`int_union`
$$ \frac{\Gamma \vdash e_1 : \text{set of int} \quad \Gamma \vdash e_2 : \text{set of int}}{\mathcal{D}(e_1 \cup e_2) = \mathcal{D}(e_1) \cup_i \mathcal{D}(e_2)} $$

`float_union`
$$ \frac{\Gamma \vdash e_1 : \text{set of float} \quad \Gamma \vdash e_2 : \text{set of float}}{\mathcal{D}(e_1 \cup e_2) = \mathcal{D}(e_1) \cup_f \mathcal{D}(e_2)} $$

Intersection

`int_intersection`

$$ \frac{\Gamma \vdash e_1 : \text{set of int} \quad \Gamma \vdash e_2 : \text{set of int}}{\mathcal{D}(e_1 \cap e_2) = \mathcal{D}(e_1) \cap_i \mathcal{D}(e_2)} $$

`float_intersection`
$$ \frac{\Gamma \vdash e_1 : \text{set of float} \quad \Gamma \vdash e_2 : \text{set of float}}{\mathcal{D}(e_1 \cap e_2) = \mathcal{D}(e_1) \cap_f \mathcal{D}(e_2)} $$

Mixed
$$ \frac{\Gamma \vdash e_1 : \text{set of int} \quad \Gamma \vdash e_2 : \text{set of float}}{\mathcal{D}(e_1 \cap e_2) = \text{set\_i2f}(\mathcal{D}(e_1)) \cap_f \mathcal{D}(e_2)} $$

Difference

`int`

$$ \frac{\Gamma \vdash e_1 : \text{set of int} \quad \Gamma \vdash e_2 : \text{set of int}}{\mathcal{D}(e_1 \setminus e_2) = \mathcal{D}(e_1) \setminus_i \mathcal{D}(e_2)} $$

`float`
$$ \frac{\Gamma \vdash e_1 : \text{set of float} \quad \Gamma \vdash e_2 : \text{set of float}}{\mathcal{D}(e_1 \setminus e_2) = \mathcal{D}(e_1) \setminus_f \mathcal{D}(e_2)} $$

Mixed
$$ \frac{\Gamma \vdash e_1 : \text{set of int} \quad \Gamma \vdash e_2 : \text{set of float}}{\mathcal{D}(e_1 \setminus e_2) = \text{set\_i2f}(\mathcal{D}(e_1)) \setminus_f \mathcal{D}(e_2)} $$
$$ \frac{\Gamma \vdash e_1 : \text{set of float} \quad \Gamma \vdash e_2 : \text{set of int}}{\mathcal{D}(e_1 \setminus e_2) = \mathcal{D}(e_1) \setminus_f \text{set\_i2f}(\mathcal{D}(e_2))} $$
